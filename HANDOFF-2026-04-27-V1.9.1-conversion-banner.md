## 1. Contexte

Suite à V1.9-transcript-persistance (Copilot LIVE stable + bannière flottante de base), MH demande la transformation de la bannière en **assistant de closing temps réel** orienté conversion : carousel suggestions, jauges intelligentes par intention, CTA dynamique contextuel, allègement colonne droite, et bannière persistante auto-ouverte.

5 itérations UX successives validées par MH :
1. Design premium glassmorphism (refus tel quel — scope élargi)
2. Allègement colonne droite + carousel + jauges + Raccrocher
3. CTA "Accepter" générique → CTA dynamique avec labels par intention
4. Détection intention dans le texte (kind='analysis' sans tag) + persistance bannière + simplification wording
5. Auto-open à chaque appel

### B — Bannière flottante = centre d'action (CollabPortal.jsx)

#### B.1 — Design premium glassmorphism
- `backdrop-filter: blur(24px) saturate(180%)`
- Gradient 3-stops blanc→bleuté→violet (rgba 0.84/0.88/0.86)
- `border-radius: 20px 20px 0 0`, ombre double-layer `0 -14px 48px rgba(124,58,237,0.16)`
- 3 zones : (1) Statut Copilot+Timer+Contact (compact, gauche, séparateur droit) | (2) Carousel suggestion (centre, dominant) | (3) CTA dominant + secondaires + Raccrocher + close

#### B.2 — Carousel (1 slide visible)
- Source : `phoneLiveAnalysis` (suggestion principale GPT) + `phoneLiveSuggestions.filter(pending)` (signaux atomiques)
- Navigation manuelle ‹ › avec indicateur `N/M`. Pas d'auto-rotation (non-stressant)
- Animation `cockpitSlideEnter 0.32s` à chaque changement
- Empty state : 3 dots loading staggered + "L'IA analyse l'appel…"

#### B.3 — Jauges intelligentes (4 boutons)
- States : `cockpitIntentLevels {accept, rdv, sms, email}` (cap 0..4)
- useEffect watche `phoneLiveSuggestions` — incrément à chaque nouvel item selon mapping :
  - `rdv|rappel|disponibilite` → bouton **RDV**
  - `document|devis|email_dicte` → bouton **Email**
  - `accord|interet|satisfaction|qualification` → bouton **Accepter**
  - `canal` → bouton **SMS**
- Visuel : overlay vertical interne `position:absolute; bottom:0; height:${level*25}%` qui monte de bas en haut (transition 0.6s cubic-bezier)
- Niveau 3+ : `cockpitGaugeReady` ring pulse (signal "action prioritaire")
- Niveau 4 : vibration unique 0.5s (`cockpitGaugeVibrate`) déclenchée 1x via `cockpitVibrationFiredRef`
- Reset : à fin d'appel OU au clic du bouton (`consumeIntent(key)`)
- Title tooltip dynamique : "Action prioritaire — RDV demandé plusieurs fois" quand level ≥ 3

#### B.4 — CTA dynamique contextuel
**Mapping par TAG** (slide.kind='detect', tag présent) — 7 mappings :
| Tag | Label | Action |
|---|---|---|
| `rdv` / `rappel` / `disponibilite` | **Créer le RDV** | `setPhoneScheduleForm` pré-rempli + ouvre modal |
| `document` / `devis` | **Envoyer la plaquette** | `mailto:` avec subject "Documentation suite à notre échange" |
| `email_dicte` | **Envoyer email** | regex extract email du texte slide → mailto |
| `canal` / `tel_dicte` | **Envoyer un SMS** | `setPhoneRightTab('sms')` + dépliage panel |
| `accord` | **Passer en client** | `pipeline_stage:'client_valide'` |
| `interet` / `qualification` / `satisfaction` / `besoin` | **Marquer comme intéressé** | `pipeline_stage:'qualifie'` |
| `objection` / `insatisfaction` | **Ajouter une objection** | append timestamped dans `notes` du contact |
| autres types tag | **Enregistrer l'idée** | append dans `notes` |

**Détection intention dans TEXTE** (slide.kind='analysis', pas de tag) — helper `detectIntentFromText`. Patterns français :
- **rdv** : `rendez-vous`, `rdv`, `créneau`, `disponibilité`, `prendre date`, `on se voit`, `fixer`, `programmer`, `planifier`, `bloquer`, `réserver`
- **sms** : `sms`, `texto`, `whatsapp`, `message court`
- **email** : `plaquette`, `brochure`, `documentation`, `devis`, `envoyer par mail`, `envoyez-moi`, `je vous envoie`, `par mail`/`par email`
- **accord** : `je signe`, `j'accepte`, `on y va`, `c'est validé`, `c'est conclu`, `je dis oui`, `d'accord`, `c'est bon pour moi`, `je prends`
- **qualif** : `qualifier`, `intéressé`, `son besoin`/`ses besoins`, `son budget`, `son projet`
- **objection** : `objection`, `trop cher`, `pas intéressé`, `réfléchir`, `hésit`

**Catch-all** = "Enregistrer l'idée" → append dans notes contact (action concrète, pas un log opaque).

#### B.5 — Bouton Raccrocher
Rouge gradient `#EF4444 → #B91C1C`, padding 12×16, box-shadow rouge, hover plus foncé. Appelle `endPhoneCall()` avec try/catch.

#### B.6 — Bannière persistante (règle MH)
- **Aucun `setCockpitOpen(false)` automatique** dans les actions CTA
- Fermeture **exclusivement** : croix `X` utilisateur OU fin d'appel (useEffect reset existant)

#### B.7 — Auto-open à chaque appel
Nouveau useEffect [CollabPortal.jsx:1424-1431](app/src/features/collab/CollabPortal.jsx#L1424) :
```js
useEffect(() => {
  if (phoneActiveCall) {
    setCockpitOpen(true);
    setCockpitMinimized(false);
  }
}, [phoneActiveCall]);
```
- Démarrage appel (entrant ou sortant) → bannière s'ouvre auto
- Indépendant du contact sélectionné, pipeline, states IA
- Cohabite avec le reset à fin d'appel (mutuellement exclusifs)
