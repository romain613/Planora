## 3. Flow restore (sécurité critique)

```
1. readSnapshot(snapshotId)
   ├─ Vérification SHA-256 (intégrité gzip)
   ├─ Validation schema version
   └─ Throw immédiat si corruption / version non supportée

2. computeCollabFingerprint() → beforeFingerprint

3. buildCollabSnapshot() + writeSnapshot() avec :
   ├─ kind='pre-restore'
   ├─ trigger='restore-reversibility'
   ├─ createdBy='<actorType>:<actorId>'
   └─ expiresAt = now + 7 jours (immunité retention auto)

4. db.transaction(() => {
     PRAGMA defer_foreign_keys = ON  -- check FK différé jusqu'au COMMIT

     // DIRECT_TABLES write-safe (15 tables)
     for each table:
       DELETE FROM <table> WHERE <scope-collab>
       for each row in snapshot[table]:
         INSERT OR REPLACE INTO <table> (...) VALUES (...)

     // CONTACT_JOINED_TABLES write-safe (4 tables)
     // DELETE par contactIds chunked (limite 800 placeholders SQLite)
     // INSERT idem

     // SKIP : read-only-v2 tables (call_logs, transcripts, archive, activity_logs)
   })  -- COMMIT (FK validés ici) ou ROLLBACK auto sur erreur

5. computeCollabFingerprint() → afterFingerprint

6. INSERT audit_logs (immutable trigger ABORT update/delete)
   ├─ entityType: 'collab_snapshot'
   ├─ action: 'collab_snapshot_restored'
   ├─ category: 'data-recovery'
   ├─ metadata_json: snapshotId, preRestoreSnapshotId, before/afterFingerprint, restored counts, skipped, txElapsedMs

7. UPDATE collaborators SET dirtySinceSnapshotAt = NULL WHERE id = ?
   (état aligné sur snapshot connu, pas besoin de re-snapper immédiatement)
```

**Garanties** :
- ✅ Scope STRICT au collabId (jamais de leak vers autres collabs)
- ✅ Tables OUT_OF_SCOPE jamais touchées (audit_logs, sessions, sms, voip credits, google_events, settings entreprise...)
- ✅ Pre-restore obligatoire avant apply (réversibilité 7 jours)
- ✅ Transaction atomique : ROLLBACK auto sur erreur
- ✅ FK constraints respectées via `defer_foreign_keys`
- ✅ Audit_logs immutable (trigger SQLite ABORT)

## 10. Routes API exposées (référence)

### GET /api/collab-snapshots/list
Query : `collabId` (optionnel pour self) | `companyId` (optionnel admin) | `limit` (défaut 50, max 200)
Auth : self ou admin

### GET /api/collab-snapshots/:id/preview
Auth : self ou admin (vérifie collabId/companyId du snapshot)

### POST /api/collab-snapshots/:id/restore
Body : `{ confirmed: true, reason?: string }` — `confirmed:true` obligatoire (anti-clic accidentel)
Auth : self ou admin
Effet : crée pre-restore + applique en transaction + audit_logs immutable

### POST /api/collab-snapshots/manual
Body : `{ collabId?: string, companyId?: string (admin), reason?: string }`
Auth : self ou admin
Effet : force un snapshot kind='manual' / trigger='user-manual' (pas d'expiration)
