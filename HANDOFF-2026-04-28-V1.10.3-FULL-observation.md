## 2. Chaîne complète Reporting Collab RDV

### Source de vérité (SCOPE STRICT)

```
sender   = bookings.bookedByCollaboratorId
receiver = bookings.agendaOwnerId
scope    = bookings.bookingType = 'share_transfer'
```

### Backend P1 (commit `15a29d44`)

- `GET /api/bookings` filtre élargi : `(collaboratorId OR agendaOwnerId OR bookedByCollaboratorId)`
- `PUT/DELETE /api/bookings/:id` ownership élargi : sender|receiver|admin|supra
- 4 colonnes ALTER : `bookingReportingStatus`, `bookingReportingNote`, `bookingReportedAt`, `bookingReportedBy`
- 2 indexes : `idx_bookings_reporting_status`, `idx_bookings_reported_by`

### Backend P2 (commit `15a29d44`)

- `GET /api/bookings/reporting?role=received|sent` (scope strict share_transfer)
- `PUT /api/bookings/:id/report` : 8 contrôles (company, scope, auth, anti-double, enum, note, audit, notif)
- Enum 7 statuts : `pending/validated/signed/no_show/cancelled/follow_up/other`
- Note obligatoire pour : `signed/cancelled/no_show/follow_up/other`
- Audit log immutable `action='booking_reported', category='rdv_reporting'`
- Notification au sender via `createNotification` helper

### Frontend P3 (commit `15a29d44` + `41fe7ea8`)

- `RdvReportingTab.jsx` : tab "📊 Reporting RDV" avec 2 sous-onglets Reçus/Transmis + modal 6 statuts
- `FicheReportingBlock.jsx` : bloc dans fiche client (CrmTab.jsx avant sub-tabs) — sender/receiver/admin views

### Backend FIX P1-full (commit `41fe7ea8`)

- POST /api/bookings auto-marque `bookingType='share_transfer'` si `bookedByCollaboratorId !== collaboratorId`
- Préserve explicit `bookingType` (ex: `internal` interMeetings)
- Backfill SQL : 6 bookings legacy `external` → `share_transfer`

### Frontend Pipeline Live P2-full (commit `41fe7ea8`)

- Card border orange `#F97316` si `_isSharedCard` (parsing shared_with + shared_with_json)
- Badge wording "Apporté à X" → "Transmis à X" (orange)
- Sous-texte "⏳ Reporting en attente" si booking pending
- Drag/select disabled non-owner (déjà V1.8.14)

### Frontend fiche P3-full (commit `41fe7ea8`)

- Composant isolé `FicheReportingBlock.jsx` (170 lignes)
- Position : juste avant les sub-tabs de la fiche modal
- Visibilité auto-masquée si rien à afficher
- Header role-aware : "Transmis à X" / "Reçu de X" / "X → Y" (admin)
- Bouton "Faire le reporting" receiver-only → nav `rdv-reporting` tab
