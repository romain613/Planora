## 4. RÈGLES CRITIQUES (à respecter absolument)

### Architecture

1. **Structure cible** :
   - `shared/ui/` = composants UI atomiques (réutilisables partout)
   - `shared/utils/` = fonctions pures sans state
   - `shared/services/` = wrappers API
   - `shared/state/` = état global volatile (tabState)
   - `features/<domaine>/` = code métier par domaine
   - `features/<domaine>/screens/` = écrans métier (HookIsolator screens)
   - `features/<domaine>/tabs/` = tabs extraits via context
   - `features/<domaine>/context/` = contexts par domaine
   - `features/<domaine>/forms/` = formulaires métier
   - `features/<domaine>/data/` = constantes/templates métier

2. **Ne PAS remettre de code dans App.jsx** ❗ App.jsx est un **orchestrateur pur** (605 l). MH a explicitement demandé de **toujours** structurer le nouveau code dans la bonne hiérarchie. Si on ne sait pas où mettre un nouveau composant, demander avant.

3. **Les tabs extraits consomment le context** via `useCollabContext()` — pas de prop drilling.

4. **Pour étendre un tab existant** : modifier le fichier dédié (`features/collab/tabs/MessagesTab.jsx` etc.), pas dans CollabPortal.

5. **Pour ajouter un nouveau tab collab** :
   1. Créer `features/collab/tabs/NewTab.jsx` qui consomme le context
   2. Étendre `CollabContext` value dans CollabPortal si nouveaux refs nécessaires
   3. Ajouter `import NewTab from "./tabs/NewTab"` dans CollabPortal
   4. Ajouter `{portalTab === "new-tab" && <NewTab/>}` dans le JSX render

### Règles de refactor

6. **Une extraction = un patch = un commit = un deploy** (jamais chaîner plusieurs phases en un push)
7. **Backup auto avant modification** — utiliser `App.jsx.pre-<phase>-<ts>` ou équivalent
8. **Rollback prêt avant chaque deploy** — chaque commit est revertable individuellement
9. **Si extraction nécessite > 10 props → stop**, étendre le context d'abord
10. **Si un composant a > 1 500 lignes après extraction → subdiviser**

### Règles de build/deploy

11. **Jamais `--minify=false`** sur le VPS (OOM 3+GB RAM)
12. **Toujours copier dist → httpdocs** après `npm run build`
13. **Vérifier HTTP 200 + bundle hash** après deploy

### Backend (§0 de CLAUDE.md — règle absolue)

14. **Jamais `import { db } from '../db/database.js'` direct** dans les routes backend. Toujours via `tenantResolver`.
15. **Chaque nouvelle entreprise = sa propre DB SQLite** dans `/var/www/planora-data/tenants/`
